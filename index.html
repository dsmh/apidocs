API de Gestión de Empleados y Posiciones (1.0.0)

Download OpenAPI specification:

Propuesta de API para la gestión de empleados, validación de disponibilidad y creación de posiciones desde el sistema Transkal. Esto es un esquema a tipo de borrador de lo que se pretende construir. La API Key será generada por nosotros luego de construir y desplegar esta API.

Empleados

Crear nuevo empleado

Endpoint para registrar un nuevo empleado en el sistema. Se debe enviar primero al sistema el mapa de cargos.

Authorizations:
ApiKeyAuth
Request Body schema: application/json
required
email
required
string <email>

Correo electrónico del empleado.

name
required
string

Nombre completo del empleado.

identification
required
string

NIF o DNI del empleado.

positionName
string

Nombre del cargo o posición. Esta debe haber sido previamente creada, con el endpoint que se encuentra en apartado de posiciones, aquí documentado.

contractDate
required
string

Fecha de inicio del contrato o inicio de labores en la empresa. Vital para calular los periodos de vacaciones (Formato -> YYYY-MM-DD).

objectType
string

Tipo de objeto en el software externo. Útil para identificar el tipo de objeto al que se hace referencia en el sistema "Padre" (El sistema que esté creando en usuario). (¿Se será necesario? ¿Con el externalId ya no se podría deducir el tipo de objeto?)

externalId
string

ID del objeto en el software externo.

superiorPositionName
required
string

Nombre del cargo superior inmediato, es opcional, pero si se desea crear un organigrama, es obligatorio. Debe haber sido previamente creado con el endpoint de posiciones. example -> Líder Técnico

phoneNumber
string

Número de teléfono de contacto (opcional).

example
any

Posiciones

Crear posiciones (Carga masiva)

Endpoint para crear una o múltiples posiciones. Acepta un arreglo de objetos de posición. Este endpoint debe ser ejecutado antes de crear un nuevo empleado, ya que es el que genera el mapa de cargos necesario para construir el organigrama.

Authorizations:
ApiKeyAuth
Request Body schema: application/json
required
Array
positionName
required
string

Nombre del nuevo cargo.

superiorPositionName
string

Nombre del cargo superior, Obligatorio, solo será null en el caso del gerente general. Esto es prioritario para construir el organigrama.

areaName
required
string

Nombre del área al que pertenece el cargo.
